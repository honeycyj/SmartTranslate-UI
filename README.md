# Menu Design Lab

一个基于 `Vite + React + TypeScript` 的菜单交互 Demo，用于展示说话人标签菜单的视觉和交互方案。

当前仓库更适合作为开发接手用的原型工程，而不是 CodePen 片段。开发可以直接拉取后本地运行、继续拆组件，或接入真实业务数据。

## 技术栈

- React 19
- TypeScript
- Vite

## 本地启动

```bash
npm install
npm run dev
```

默认本地开发地址由 Vite 提供，通常是 `http://localhost:5173`。

生产构建：

```bash
npm run build
```

## 项目结构

```text
.
├── public/assets/menu/        # 菜单图标与头像资源
├── src/App.tsx                # 主交互与页面结构
├── src/main.tsx               # 应用入口
├── src/styles.css             # 全部样式
├── index.html                 # Vite 页面入口
└── package.json               # 依赖与脚本
```

## 已实现内容

- 菜单主体 UI 和视觉样式
- 说话人条目 hover 态
- 说话人选中态
- 右侧更多操作菜单定位与显示
- “合并说话人”流程
- 拖拽说话人进入合并区
- 合并候选顺序调整
- 点击“合并”后合并标签显示

## 当前交互说明

页面打开后可以直接体验以下交互：

1. 点击任一说话人行，可切换选中状态。
2. hover 某一行后，右侧会显示更多按钮。
3. 点击更多按钮，可打开操作菜单。
4. 选择“合并说话人”后，会进入合并模式。
5. 将其他说话人拖入合并区后，点击“合并”可完成合并。

## 当前未接入业务的部分

以下内容目前是原型态或占位态，开发接手时需要按业务补齐：

- “重命名”菜单项目前没有实际逻辑，只保留了入口。
- 顶部“当前剧集 / 全剧”是静态展示，没有切换数据。
- 底部“新增说话人标签”按钮目前没有事件处理。
- 数据全部为前端内置 mock，没有接真实接口。
- 合并结果仅保存在组件本地状态，刷新页面后不会持久化。

## 开发接入建议

如果要把这个 Demo 接入正式项目，建议按下面方式处理：

1. 把 `src/App.tsx` 中的本地状态替换为业务数据源。
2. 将说话人行、标签、上下文菜单、合并卡片拆成独立组件。
3. 将 `initialSpeakerRows` 替换为接口返回的数据结构。
4. 为“重命名”“新增标签”“标签切换”补充真实交互和接口。
5. 根据目标项目的设计系统，把 `src/styles.css` 中的样式迁入对应的样式方案。

## 资源说明

- 图标和头像资源位于 `public/assets/menu`
- 代码中使用的是站点根路径资源引用，例如 `/assets/menu/icon-check.svg`
- 在 Vite 本地开发和构建产物中可直接使用
- 如果迁移到其他托管方式，需要确认静态资源基础路径是否一致

## 适合作为 GitHub 交付的原因

- 结构完整，可直接运行
- 开发无需从在线 demo 反向还原工程
- 便于继续拆分组件、接接口、补测试
- 比 CodePen 更适合作为 handoff 版本

## 备注

这是一个偏交互原型性质的仓库，重点在 UI 行为表达，不包含后端、持久化和业务数据管理。
